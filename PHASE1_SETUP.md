# JobCatch — Phase 1: Database & Authentication

This is the backend foundation: the full data schema, a custom email-based `User`
with per-role profiles, and a complete JWT auth layer (register → verify → login →
refresh → logout) with RBAC and throttling.

---

## What's in this phase

**All 17 database entities** across 11 apps (`accounts`, `catalog`, `bookings`,
`payments`, `subscriptions`, `jobs`, `reviews`, `messaging`, `notifications`,
`verification`, `common`) — UUID PKs, PostGIS geography fields, and the `pgvector`
extension enabled for the Phase 3 AI chatbot.

**Authentication** — SimpleJWT with refresh rotation + blacklist, a `token_version`
revocation mechanism (powers logout-all / forced logout on password reset), OTP email
verification + password reset, scoped throttling on login/OTP endpoints, and
`SessionRecord`-backed device management.

**RBAC** — role permission classes (`IsCustomer/IsArtisan/IsEmployer/IsJobSeeker/IsAdmin`)
plus an object-ownership base, with role read from the DB user (not the raw token).

## Run it

```bash
# 1. Config
cp .env.example .env            # fill SECRET_KEY + JWT_SIGNING_KEY at minimum

# 2. Bring up Postgres(PostGIS) + Redis + backend + celery
docker compose up --build -d db redis
docker compose run --rm backend python manage.py migrate      # see migration order below
docker compose run --rm backend python manage.py seed_demo
docker compose run --rm backend python manage.py createsuperuser   # optional (seed makes one)
docker compose up backend celery
```

Without Docker (local Python): install GDAL/GEOS/PROJ, `pip install -r requirements.txt`,
point `DATABASE_URL` at a PostGIS DB, then `python manage.py migrate && seed_demo && runserver`.

### Migration order (important)
`apps/common/migrations/0001_enable_extensions.py` creates the `postgis` and `vector`
extensions and uses `run_before` to force itself ahead of every geo-using app's first
migration. So the flow is simply:

```bash
python manage.py makemigrations     # generates model migrations for all apps
python manage.py migrate            # 0001_enable_extensions runs first automatically
```

On Render (managed Postgres), enable the PostGIS + vector extensions from the dashboard;
the `CreateExtension` operations then become harmless no-ops.

## Auth endpoint surface (`/api/v1/auth/`)

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| POST | `register/` | Create account (role ≠ admin), send email OTP | public |
| POST | `verify-otp/` | Confirm email → auto-login (returns tokens) | public, throttled |
| POST | `resend-otp/` | Re-send a code | public, throttled |
| POST | `login/` | Email+password → tokens (blocked until verified) | public, throttled |
| POST | `token/refresh/` | Rotate tokens (cookie on web / body on mobile) | public |
| POST | `logout/` | Revoke current device (blacklist + drop session) | authed |
| POST | `logout-all/` | Bump token_version → invalidate every token | authed |
| POST | `password-reset/request/` | Send reset OTP | public, throttled |
| POST | `password-reset/confirm/` | Set new password + force re-login | public, throttled |
| GET | `me/` | Current user + role profile | authed |
| GET/DELETE | `sessions/` , `sessions/{id}/` | List / revoke active sessions | authed |

Interactive docs at `/api/docs/` (Swagger) and schema at `/api/schema/` (drf-spectacular).

### Token placement recap
- **Web (React):** refresh token in an **HttpOnly Secure SameSite cookie**; access token held in memory.
- **Mobile (Expo):** both tokens in **expo-secure-store**; the refresh is returned in the JSON body for that purpose.

## Definition of Done — verification checklist

- [ ] `migrate` applies cleanly (extensions first, then all model tables).
- [ ] `seed_demo` creates 10 categories + one user per role + a superuser.
- [ ] Register an artisan → OTP prints to console (dev email backend) → `verify-otp` returns tokens.
- [ ] `GET /me/` with the access token returns the user **and** the nested artisan profile.
- [ ] Login is refused with `email_unverified` before verification.
- [ ] `logout-all/` makes a previously-valid access token return 401 (token_version check).
- [ ] `pytest` passes (`apps/accounts/tests.py` covers the above + RBAC).
- [ ] Django admin at `/admin/` lists all 17 entities.
- [ ] Swagger UI renders the auth endpoints at `/api/docs/`.
